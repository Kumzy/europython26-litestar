# litestar-vite - the contract reaches the frontend

::left::

```python
from litestar_vite import (
    PathConfig, TypeGenConfig, ViteConfig, VitePlugin,
)

vite = VitePlugin(config=ViteConfig(
    mode="spa",                  # template · htmx · inertia · …
    dev_mode=settings.DEV_MODE,  # HMR, proxied through Litestar
    paths=PathConfig(
        root="src/js/web",
        bundle_dir="app/server/static/web",
        asset_url="/static/web/",
    ),
    types=TypeGenConfig(output="src/lib/generated"),
))
```

<ul class="stars mt-3">
<li class="muted">Dev: <code>litestar run</code> starts Vite alongside the app</li>
<li class="muted">Typegen emits <code>openapi.json</code>, <code>routes.ts</code>, <code>schemas.ts</code></li>
<li class="muted">Prod: <code>litestar assets build</code> → hashed bundles + manifest</li>
</ul>

::right::

```ts
// AUTO-GENERATED by litestar-vite - do not edit
import { route } from '@/lib/generated/routes'
import type { Tag } from '@/lib/generated/schemas'
// openapi.json → typed SDK + query hooks (hey-api)
import { listTags } from '@/lib/generated/api/sdk.gen'

const url = route('list_tags') // names are a union type
const page = await listTags() // request & response typed
```

<ul class="stars mt-3">
<li><strong>The DTO from slide 19 doesn't stop at the wire</strong> - the same declared shape drives OpenAPI, docs, and the generated client</li>
<li class="muted">Rename a DTO field → the frontend fails to <em>compile</em>, not at runtime</li>
</ul>
