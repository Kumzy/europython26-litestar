# SQLSpec - explicit SQL, same handler shape

::left::

```sql
-- db/sql/users.sql
-- name: list-users
SELECT id, email, name, is_active, created_at
FROM user_account
ORDER BY created_at DESC
LIMIT :limit OFFSET :offset;
```

```python
class UserService(SQLSpecAsyncService):
    async def list_with_count(
        self, *filters: StatementFilter
    ) -> OffsetPagination[User]:
        return await self.paginate(
            db_manager.get_sql("list-users"),
            *filters,
            schema_type=User,  # msgspec Struct
        )
```

<ul class="stars mt-3">
<li class="muted">The SQL that runs is the SQL you read - a file you can review and <code>EXPLAIN</code></li>
<li class="muted">Typed results without an ORM; 18+ adapters (Postgres, DuckDB, Oracle, BigQuery…)</li>
</ul>

::right::

```python
def provide_users_service(db_session: AsyncpgDriver) -> UserService:
    return UserService(db_session)  # session: plugin-provided

class UserController(Controller):
    path = "/api/users"
    dependencies = create_filter_dependencies({
        "search": "name,email",
        "pagination_type": "limit_offset",
    }) | {"users_service": Provide(provide_users_service)}

    @get(operation_id="ListUsers")
    async def list_users(
        self,
        users_service: UserService,
        filters: Annotated[list[FilterTypes],
                           Dependency(skip_validation=True)],
    ) -> OffsetPagination[User]:
        return await users_service.list_with_count(*filters)
```

<ul class="stars mt-3">
<li><strong>Same handler shape as the ORM path</strong> - swap the data layer, keep the route</li>
<li class="muted">One manager, many databases · Arrow-native results</li>
</ul>
